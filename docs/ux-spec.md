# UX 交互规范 — mind-sync

> 本文档描述每页面在不同状态下的交互行为，**区别于 PRD 的功能描述**。
> PRD 定义 WHAT，本文档定义 HOW。

---

## 1. 导航菜单

### 1.1 基本行为

| 规则 | 描述 |
|------|------|
| 唯一活性 | 同一时刻只有一个父级菜单 / 子级菜单处于选中状态 |
| 切换展开 | 切换到有子级的父级菜单时，展开该父级 + 选中第一个子级（或是页面则显示） |
| 切换折叠 | 从有子级的父级 A 切换到父级 B 时，A 自动折叠，B 展开 |
| 账户固定 | 「账户」始终在侧边栏底部，不参与菜单切换 |

### 1.2 文档库特殊规则

| 场景 | 行为 |
|------|------|
| 初次打开 | 无子级被选中、所有子级折叠、显示「选择左侧文件开始阅读」|
| 非初次（同次登录） | 保留上次展开的树结构 + 选中的子级/文档 |
| 跨登录 | 同上，状态持久化到 localStorage |

## 2. 页面保持

| 场景 | 行为 |
|------|------|
| 页面切换（切走再切回） | 保留前回显示内容 + 状态 |
| 登录/登出 | 保留（登出不清除，登录后恢复） |
| F5 刷新 | 保留 |
| Ctrl+F5 强制刷新 | 保留 |
| 服务器重启 | 不保留（可选优化，当前无需求） |

## 3. 弹窗

| 规则 | 描述 |
|------|------|
| 使用方式 | 全部用 Vue 组件式（禁止 alert/confirm/prompt）|
| 关闭方式 | 点击遮罩层 / ESC 键 |
| 位置 | 居中 |
| 覆盖 | 显示期间页面不可操作 |

## 4. 搜索页面

### 4.1 输入与搜索

| 操作 | 行为 |
|------|------|
| 输入关键字 → 点击搜索按钮 | 执行搜索 |
| 输入关键字 → 回车 | 同点击搜索 |
| 空输入 → 点击搜索 | 清空结果，显示「无结果」友好提示 |
| 搜索按钮右测「重置」按钮 | 有结果时显示，点击后清空结果回到初始页 |

### 4.2 筛选与排序

| 操作 | 行为 |
|------|------|
| 切换排序方式（相关度/最近修改） | 自动过滤排序，无需重新搜索 |
| 切换分类（全部/原始素材/学习摘要/问答沉淀） | 自动过滤，无需重新搜索 |

### 4.3 搜索结果

| 场景 | 行为 |
|------|------|
| 0 件 | 显示「无结果」友好提示 |
| 多件 | 分页（每页 5 件），标明文档来源 |
| 关键字高亮 | 搜索结果中高亮匹配关键字 |

### 4.4 搜索结果 → 文档跳转

| 操作 | 行为 |
|------|------|
| 单击搜索结果 | 跳转至文档库并打开文档 |
| 文档内高亮 | 所有搜索关键字高亮 |
| 首个高亮居中 | 自动滚动使第一个高亮在屏幕居中 |
| 键盘 ↑↓ | 切换上一个/下一个高亮关键字 |
| 侧边栏树 | 自动展开并定位到匹配文档 |

## 5. 文档库页面

### 5.1 文档显示

| 文档类型 | 显示行为 |
|----------|----------|
| Markdown (.md) | 正确渲染：表格、图片、链接、代码块 |
| 代码 (.py/.java/…) | 语法高亮（语言识别 + 颜色标记） |

### 5.2 搜索结果高亮

| 场景 | 行为 |
|------|------|
| 当前文档在搜索结果列表中 | 所有关键字高亮 |
| 右侧「重置搜索结果」按钮 | 可见，点击后清除所有高亮、恢复键盘原始功能、保持文档打开 |

### 5.3 键盘操作

| 场景 | ↑↓ 行为 |
|------|----------|
| 从搜索跳转的文档 | 切换上一个/下一个高亮关键字 |
| 手动打开文档（在搜索结果中）| 同上 |
| 手动打开文档（不在搜索结果中）| 上下滚动页面 |
| 从其他页面切回文档库 | 根据前回显示的文档行为恢复 |

## 6. 页面权限

| 场景 | 行为 |
|------|------|
| 非管理员访问管理页面 | 跳转到文档库 + 黄底提示「页面『xxx』仅管理员可访问」 |
| 管理员专属菜单 | 完全不显示（非管理员不可见） |
